## 2026-06-15

### 本轮方案决策

| 项目 | 结论 |
|---|---|
| 检测框架 | `MMYOLO`（基于 `MMEngine/MMDetection` 生态） |
| 代码起点 | 从零搭建 |
| Backbone 初始化 | RGB / Depth / Thermal 三路都用 ImageNet 权重初始化 |
| Depth 预处理 | `16bit -> [0,1] -> log(raw_d + 1) / log(20000)` |
| Depth 无效区域 | 保留 `0` 值，不做 mask，不做补全 |
| Thermal / Depth 输入通道 | 单通道复制成 `3` 通道 |
| 模态内增强 | 完整复现 M2I2HA 的 `FuseSEBlock` + `LR-C3AH` + `DSC3k` |
| 质量评估 | 方案 A + B 都做（`σ_i` + `q_i`） |
| Pretext 损坏策略 | 在线随机损坏：高斯噪声 / 随机块置零 / 坏像素 / 对比度压缩 |
| Quality Head 训练 | 先 pretext 预训练，再与检测任务联合训练，保留 BCE 约束 |
| 三模态 Inter-Hypergraph | 一套超边同时连接 RGB / Depth / IR，不做三套两两配对 |
| 最终提交置信度 | 直接使用融合检测头输出的 `cls_score` |
| 模态独立辅助置信度 | 在 GateFusion 前挂轻量辅助头输出 `c_rgb` / `c_d` / `c_t`，仅用于诊断与辅助监督 |
| 验证集切分 | `9:1`（`1800 / 200`） |
| NMS | 先沿用 YOLOv8 / MMYOLO 默认阈值，后续在验证集调参 |

### 关键理由

- Depth 样例中 `0` 值大面积存在，且真实目标常落在 `depth = 0` 区域；直接 mask 或补全会破坏这种真实传感器退化信号。
- 单通道复制成 `3` 通道可以保持第一层 `Conv2d(3, ...)` 结构不变，直接复用 ImageNet 低层边缘/纹理特征。
- 质量信号已经在融合阶段参与权重分配，最终 `confidence` 不再额外乘质量因子，避免对低质量模态重复惩罚。
- 三模态 Inter-Hypergraph 采用“一套三角连接超边”即可，同时建模三个模态的高阶关系，结构比三套两两配对更干净。
- Quality Head 联合训练时保留 pretext BCE，可以避免质量头完全漂移成“只对 detection loss 有利”的代理分数。
